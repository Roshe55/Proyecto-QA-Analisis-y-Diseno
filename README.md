# Proyecto-QA: Urban Routes (Análisis y Diseño)

## 📋 Descripción del Proyecto
Este proyecto se enfocó en el análisis crítico de requisitos y el diseño de pruebas estructuradas para la funcionalidad de "compartir automóvil" (carpool) en la plataforma **Urban Routes**. El objetivo fue transformar requisitos ambiguos en escenarios de prueba precisos.

## 🛠 Herramientas utilizadas
*   **Modelado:** draw.io (Mapas mentales y diagramas de flujo).
*   **Diseño de pruebas:** Google Sheets (Clases de equivalencia y análisis de valores límite).
*   **Documentación:** Google Docs (Informe final).
*   **Apoyo técnico:** IA (Generación de ideas y refinamiento de escenarios).

## 🔍 Habilidades desarrolladas
*   **Análisis de Requisitos:** Identificación de "zonas grises" e inconsistencias en la documentación funcional.
*   **Diseño de Pruebas Eficiente:** Aplicación de técnicas de caja negra (clases de equivalencia y valores límite) para optimizar la carga de trabajo sin perder calidad.
*   **Visualización:** Creación de diagramas de flujo para entender la lógica del sistema.
*   **Diseño de Casos:** Escritura de escenarios detallados (positivos y negativos) que cubren las aristas del flujo de usuario.

## 📈 Resultados
*   Documentación técnica clara que sirvió como base para la ejecución de pruebas.
*   Cobertura de pruebas robusta, asegurando que los casos críticos fueran identificados antes de la fase de ejecución.
